# Стандарт сдачи блока (чтобы заказчик НЕ проверял каждый блок руками)

> Этот документ — общий для всех сессий (session-A, session-B, …). Цель: довести
> вёрстку блока до состояния, когда заказчик даёт только **вводные** и получает
> **готовый, проверенный блок** — без ручного ревью каждого пикселя.
>
> Документ ЖИВОЙ: закончил блок — если поймал новый класс грабель, допиши его в
> раздел «Каталог граблей». Так мы обучаемся и перестаём повторять ошибки.

---

## 1. Что даёт заказчик (вводные) и что делает агент

**Заказчик даёт ТОЛЬКО:** название папки проекта · макеты (ссылки Figma с `node-id`) ·
ТЗ (`TZ/`). Всё остальное — на агенте.

**Агент делает САМ, без участия заказчика:**
- читает `CLAUDE.md`, `docs/SECTION-GUIDE.md`, `docs/COORDINATION.md`, этот файл;
- тянет дизайн (`get_design_context` / `get_screenshot`), сверяет с reference-проектами;
- верстает блок по конвенциям проекта;
- **сам сверяет рендер с макетом и сам чинит расхождения** (см. §2–§3);
- отчитывается «готово» ТОЛЬКО после прохождения чек-листа §2.

Заказчик — не инструмент сравнения «макет ↔ рендер». Это работа агента.

## 2. Definition of Done — блок готов, ТОЛЬКО если выполнено ВСЁ

Прежде чем сказать «готово», прогнать по списку. Любой ❌ = блок не готов.

- [ ] **Сборка зелёная:** `npm run build` exit 0, `validateHtml` без ошибок.
- [ ] **Скриншоты сняты** на **3+ ширинах**: мобайл `390`, десктоп `1280/1440`, и
      **ширина макета** (обычно `1920`). Playwright headless (см. §4).
- [ ] **Пиксельная сверка с макетом** (`get_screenshot` ноды ↔ наш рендер): совпадают
      цвета, отступы, размеры шрифтов, и **присутствуют ВСЕ элементы** макета
      (ничего не «пропущено»).
- [ ] **Каждая картинка реально декодируется и видна:** `naturalWidth > 0`
      (битый файл/неверное расширение → 0), элемент не `0×0`, не спрятан целиком за
      непрозрачным слоем, не нулевой `opacity` там, где должен быть виден.
- [ ] **Каждая иконка/интерактив реально отрисованы:** не «есть в DOM, но невидимы».
      Проверять вычисленные `fill`/`stroke`/`opacity`/`color`, а не только исходник.
- [ ] **Цвета берутся из вычисленного стиля**, а не «на глаз по исходнику»: золото
      вместо белого, accent вместо текста — частая ошибка темизации.
- [ ] **Адаптив:** нет горизонтального оверфлоу страницы
      (`document.scrollWidth ≤ innerWidth`) ни на одной ширине; слайдеры/таблицы
      ведут себя по конвенции (оверфлоу по ширине ЭКРАНА — см. SECTION-GUIDE);
      ничего не обрезано; mobile-first.
- [ ] **Переиспользование:** не созданы новые ассеты/компоненты/миксины там, где
      уже есть готовые (см. §3, грабли «зачем новый налепил»).
- [ ] **Чужое не сломано:** глобальные правки (`_base.scss`, `_mixin.scss`,
      `_components.scss` …) помечены в `COORDINATION.md` + приписка «Пересоберите».

**Принцип:** если элемент есть в DOM, но его не видно на скриншоте — это БАГ, а не
«готово». Скриншот, а не исходник, — источник правды.

## 3. Каталог граблей (проверять ПРОАКТИВНО — это уже стоило заказчику ревью)

Каждая строка — реальная ошибка, которую ловил заказчик. Проверяй заранее.

1. **bemto-модификаторы.** `+e.card.--bad` генерит класс `block__card--bad`.
   В SCSS селектор — `&--bad` (НЕ `&.--bad`, который = `.block__card.--bad` и не
   матчит). Симптом: модификатор «не применяется» (цвета карточек не работают).
2. **stroke-иконки в спрайте невидимы.** svgo вырезает `stroke`/`stroke-width` из
   символа → иконка в DOM есть, но без обводки невидима. Лечить в `_base.scss`:
   `.icon-<name> { fill:none; stroke:currentColor; stroke-width:<viewBox-units> }`.
   Симптом: «стрелки в кнопках нет», хотя `+icon('arrow-right')` стоит.
3. **SVG с расширением `.webp` (или наоборот).** Браузер декодирует по
   Content-Type/расширению → `naturalWidth=0`, картинка невидима. Проверять
   `file <путь>` vs расширение. Лечить переименованием + правкой `src`.
4. **Слайдер/таблица скроллится внутри `.container`, а надо по ширине ЭКРАНА.**
   Использовать `bleed-x()` / `slider-x($bleed:true)` (см. SECTION-GUIDE).
5. **Декор/фон позиционирован по `%` высоты секции** → на нашей секции (она выше
   макета из-за переносов) уезжает. Лечить фикс-геометрией (`rem`) или якорем к
   нужному краю (`bottom`/`top:50% translateY`).
6. **Ширина карточек слайдера.** Слишком узкие → не видно, что листается. Цель:
   слайд шире, но следующий чуть выглядывает (напр. `min(90vw, rem(312))`).
7. **Рамка вокруг таблицы пропадает** при технике `border` на ячейках + `margin:-1px`
   (внешний край уходит под `overflow:hidden`). Лечить gap-background (см.
   SECTION-GUIDE).
8. **«Зачем новый налепил».** Если заказчик просит «такой же, как в блоке X» —
   переиспользовать ТЕ ЖЕ файлы/компоненты/стили блока X, а не делать новый аналог.
9. **Цвет под тему.** На тёмных секциях текст/лого — белый (`--text-strong`),
   на canvas — свои токены; accent = два токена (линии/золото vs кнопки/оранж).
   Симптом: «лого бледно-жёлтое, а в макете белое» (был передан `var(--accent)`).
10. **Watermark/прозрачность.** Если элемент «бледный» — проверь `opacity` на нём
    и на родителе (бывает `opacity:0.12` на `lg+`). Цвет и прозрачность — разные вещи.
11. **Картинки — в `public/img/...` напрямую** (без пайплайна оптимизации), не в
    `source/img`.
12. **Высоту не фиксировать** px; равные высоты — через `stretch` + `height:100%`.

> Поймал новую грабли — **допиши сюда** строкой. Боковые сессии тоже дополняют.

## 4. Как верифицировать (инструменты, без участия заказчика)

Стандартный self-check — headless Playwright (`node_modules/playwright-core`):

```js
import pw from '.../node_modules/playwright-core/index.js'
const b = await pw.chromium.launch()
const p = await b.newPage({ viewport: { width: 1440, height: 1100 } })
await p.goto('http://localhost:<port>/index.html', { waitUntil: 'networkidle' })
const el = p.locator('#sBlock')
await el.scrollIntoViewIfNeeded()              // ВАЖНО: иначе скрин промахивается
await el.screenshot({ path: 'out.png' })       // → читать через Read и сравнить с макетом
// проверки:
await p.evaluate(() => document.documentElement.scrollWidth <= innerWidth + 1) // оверфлоу
await el.evaluate(n => getComputedStyle(n).color)        // реальный цвет
await img.evaluate(n => n.naturalWidth)                   // декодируется ли картинка
await ic.evaluate(n => getComputedStyle(n.querySelector('path')).fill) // видна ли иконка
```

- Снимать **локатором элемента** (а не вьюпорт-клипом) — клип промахивается, если
  секция выше вьюпорта.
- Для сравнения тянуть `mcp__claude_ai_Figma__get_screenshot` ноды и читать оба PNG.
- Сервер: `npx http-server public -p <port> -s` после `npm run build`.

## 5. Что НЕ делать

- ❌ Отчитываться «готово», не сняв и не посмотрев скриншот блока.
- ❌ Оставлять элемент в DOM невидимым (0×0 / opacity:0 / за непрозрачным слоем /
  без обводки) и считать это «свёрстано».
- ❌ Лепить новый ассет/компонент, когда есть готовый.
- ❌ Трогать глобальный файл без пометки в `COORDINATION.md`.
- ❌ Судить о цвете/наличии по исходнику — только по вычисленному стилю и скриншоту.
