## ★ Общие конвенции (соблюдать всем сессиям)
- **★★ Перед «готово» — прогнать `docs/BLOCK-DELIVERY-CHECKLIST.md` §2 (Definition of Done).** Это стандарт сдачи блока: скриншот-сверка с макетом на 3+ ширинах, проверка что все картинки/иконки реально ВИДНЫ (не только в DOM), цвета по вычисленному стилю, нет оверфлоу. Цель — заказчик НЕ проверяет каждый блок руками. **Поймал новую грабли → допиши в §3 того файла.** Когда закончите свои блоки — пройдитесь по своим правкам и тоже добавьте уроки.
- **Слайдер / широкая таблица → оверфлоу по ширине ЭКРАНА, не контейнера.** Использовать `bleed-x()` (+`bleed-x-reset()` на десктопе) из `_mixin.scss`. Подробно с примерами — `SECTION-GUIDE.md` → «Поучения» (★ ГЛАВНОЕ ПРАВИЛО СЛАЙДЕРОВ). **Никогда не оставлять слайдер скроллящимся внутри `.container`.**
- Все прочие поучения (рамка-таблица через gap-background, mobile-first, секции-канвас, акцент = 2 токена…) — там же. **Перед версткой блока — прочитать `SECTION-GUIDE.md`.**

## Правила взаимодействия сессий
- **Один блок — одна сессия.** Не редактируй `_*.pug` / `_*.scss` блока, который занят (таблица выше). Берёшь блок — добавь строку 🔓 IN PROGRESS со своим ID; закончил — ✅ DONE (или удали строку).
- **Перед каждым Edit — перечитай файл** (`Read`): соседняя сессия / владелец могли его поменять (этот файл правился параллельно даже сейчас). Правки точечные, чужое не сносить.
- Глобальные файлы (`index.pug`, `main.scss`, `_root.scss`, `_base.scss`, `_components.scss`, `_mixin.scss`, `content.json`, `common.js`) — не трогаем без явной пометки в этой таблице (см. `SECTION-GUIDE.md` → «ЗАПРЕЩЕНО»). Тронул глобальный — припиши «**Пересоберите CSS/JS**».
- После работы — `npm run build` (exit 0, `validateHtml` зелёный) + проверка скриншотами. Освободил блок — обнови таблицу.
